# LuckyFrame 用户使用说明书

## 1. 产品简介

LuckyFrame 是一个开源的自动化测试平台，旨在简化自动化测试体系建设。它采用 Web-Client 架构模式，Web 端负责信息管理与展示，Client 端负责用例执行，支持无限扩展客户端。

### 1.1 核心功能

- **分布式测试**：Web-Client 架构，支持无限扩展客户端
- **专业用例管理**：直观的自动化用例管理方式
- **质量管理**：质量数据收集分析和多图表展示
- **多线程执行**：客户端可指定线程数量，提高执行效率
- **多语言驱动**：支持 Java 和 Python 编写测试驱动
- **定时任务调度**：支持自定义配置调度任务
- **测试过程监控**：实时查看测试进度和过程
- **日志定位**：通过 LOG4J 和数据库记录测试过程
- **多纬度自动化**：支持接口、Web UI 和移动端 UI 自动化
- **HTTP+Socket 接口免编码**：纯关键字驱动，适合初级测试人员
- **在线调试用例**：直接在界面调试自动化用例
- **清新界面**：基于 Bootstrap 构建的清爽界面

## 2. 安装指南

### 2.1 环境要求

- **JDK**：1.8+
- **MySQL**：5.7+
- **Maven**：3.5+（仅开发环境需要）
- **浏览器**：Chrome、Firefox、Safari 等现代浏览器

### 2.2 安装步骤

#### 2.2.1 数据库安装与配置

1. **安装 MySQL**：下载并安装 MySQL 5.7 或更高版本
2. **创建数据库**：创建名为 `luckyframe` 的数据库
3. **导入初始化脚本**：执行 `mysqldb/initialization(create table)/V3.2 初始化.sql` 脚本初始化数据库结构
4. **执行升级脚本**：按照版本顺序执行 `mysqldb/upgrade(alter table)/` 目录下的升级脚本

#### 2.2.2 服务端安装

1. **下载安装包**：从 GitHub 或 Gitee 下载 LuckyFrameWeb 安装包
2. **配置数据库连接**：修改 `application.yml` 文件中的数据库连接信息
   ```yaml
   spring:
     datasource:
       url: jdbc:mysql://localhost:3306/luckyframe?useUnicode=true&characterEncoding=utf-8&useSSL=false
       username: root
       password: 123456
   ```
3. **启动服务**：执行 `java -jar LuckyFrameWeb.jar` 命令启动服务
4. **访问系统**：通过浏览器访问 `http://localhost:8080`

#### 2.2.3 客户端安装

1. **下载客户端**：从 GitHub 或 Gitee 下载 LuckyFrameClient 安装包
2. **配置客户端**：修改 `config.properties` 文件中的服务端地址和端口
   ```properties
   server.ip=localhost
   server.port=8080
   ```
3. **启动客户端**：执行 `startup.bat`（Windows）或 `startup.sh`（Linux）启动客户端
4. **注册客户端**：在服务端的「系统管理 -> 客户端管理」中注册客户端

## 3. 快速上手

### 3.1 登录系统

1. **访问登录页面**：打开浏览器，输入 `http://localhost:8080`
2. **输入登录信息**：默认用户名 `admin`，密码 `admin123`
3. **点击登录**：进入系统首页

### 3.2 创建测试项目

1. **进入项目管理**：点击左侧菜单「测试管理 -> 项目管理」
2. **点击新增**：填写项目名称、描述等信息
3. **保存项目**：点击保存按钮，创建测试项目

### 3.3 创建测试用例

1. **进入用例管理**：点击左侧菜单「测试管理 -> 用例管理」
2. **选择项目**：在项目下拉框中选择已创建的项目
3. **点击新增**：填写用例名称、描述、类型等信息
4. **添加用例步骤**：根据用例类型，添加相应的测试步骤
5. **保存用例**：点击保存按钮，创建测试用例

### 3.4 执行测试用例

1. **进入任务调度**：点击左侧菜单「测试执行 -> 任务调度」
2. **点击新增**：填写任务名称、选择测试用例、设置执行参数
3. **保存任务**：点击保存按钮，创建测试任务
4. **执行任务**：点击执行按钮，开始执行测试任务
5. **查看执行结果**：点击任务名称，查看测试执行结果和日志

## 6. 常见问题解答

### 6.1 登录问题

**Q：登录失败，提示用户名或密码错误**
A：请检查用户名和密码是否正确，默认用户名是 `admin`，密码是 `admin123`。

**Q：登录后无法访问某些功能**
A：请检查用户角色权限，确保用户拥有相应功能的访问权限。

### 6.2 客户端问题

**Q：客户端无法连接到服务端**
A：请检查网络连接，确保客户端和服务端在同一网络中，并且服务端地址和端口配置正确。

**Q：客户端注册失败**
A：请检查服务端是否正常运行，客户端配置是否正确，以及客户端是否已经被注册。

### 6.3 测试执行问题

**Q：测试用例执行失败**
A：请检查测试用例配置是否正确，测试环境是否正常，以及客户端是否在线。

**Q：测试执行超时**
A：请检查网络连接，测试环境响应时间，以及测试用例的超时设置。

### 6.4 数据库问题

**Q：数据库连接失败**
A：请检查数据库服务是否正常运行，数据库连接配置是否正确，以及数据库用户是否拥有相应权限。

**Q：数据库初始化失败**
A：请检查数据库版本是否符合要求，初始化脚本是否正确执行，以及数据库用户是否拥有创建表的权限。

### 6.5 其他问题

**Q：系统运行缓慢**
A：请检查服务器资源使用情况，如 CPU、内存、磁盘等，以及数据库查询性能。

**Q：系统报错**
A：请查看系统日志，定位错误原因，然后根据错误信息进行排查。

- **数据库日志**：查看 MySQL 数据库日志，了解数据库操作情况

### 7.2 常见故障及解决方法

#### 7.2.1 服务端启动失败

**症状**：执行 `java -jar LuckyFrameWeb.jar` 命令后，服务无法启动

**原因**：
- 端口被占用
- 数据库连接失败
- 配置文件错误

**解决方法**：
- 检查端口是否被占用，修改 `application.yml` 文件中的端口配置
- 检查数据库连接配置，确保数据库服务正常运行
- 检查配置文件格式，确保配置项正确

#### 7.2.2 客户端连接失败

**症状**：客户端启动后，无法连接到服务端

**原因**：
- 服务端地址或端口配置错误
- 网络连接问题
- 服务端未启动

**解决方法**：
- 检查客户端配置文件中的服务端地址和端口
- 检查网络连接，确保客户端和服务端在同一网络中
- 检查服务端是否正常启动

#### 7.2.3 测试执行失败

**症状**：测试任务执行失败，或执行结果不正确

**原因**：
- 测试用例配置错误
- 测试环境问题
- 客户端问题

**解决方法**：
- 检查测试用例配置，确保步骤和参数正确
- 检查测试环境，确保被测系统正常运行
- 检查客户端状态，确保客户端在线且资源充足

#### 7.2.4 数据库操作失败

**症状**：系统操作时提示数据库错误

**原因**：
- 数据库连接问题
- SQL 语句错误
- 数据库权限不足

**解决方法**：
- 检查数据库连接配置，确保数据库服务正常运行
- 查看系统日志，定位具体的 SQL 错误
- 确保数据库用户拥有相应的操作权限
