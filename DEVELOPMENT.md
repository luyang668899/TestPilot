# LuckyFrame 开发文档

## 1. 项目概述

LuckyFrame 是一个开源的自动化测试平台，旨在简化自动化测试体系建设。它采用 Web-Client 架构模式，Web 端负责信息管理与展示，Client 端负责用例执行，支持无限扩展客户端。

### 1.1 核心功能

- **分布式测试**：Web-Client 架构，支持无限扩展客户端
- **专业用例管理**：直观的自动化用例管理方式
- **质量管理**：质量数据收集分析和多图表展示
- **多线程执行**：客户端可指定线程数量，提高执行效率
- **多语言驱动**：支持 Java 和 Python 编写测试驱动
- **定时任务调度**：支持自定义配置调度任务
- **测试过程监控**：实时查看测试进度和过程
- **日志定位**：通过 LOG4J 和数据库记录测试过程
- **多纬度自动化**：支持接口、Web UI 和移动端 UI 自动化
- **HTTP+Socket 接口免编码**：纯关键字驱动，适合初级测试人员
- **在线调试用例**：直接在界面调试自动化用例
- **清新界面**：基于 Bootstrap 构建的清爽界面

### 2.2 系统架构

LuckyFrame 采用经典的三层架构设计：

1. **表示层（Presentation Layer）**：负责接收用户请求，展示测试结果和管理界面
2. **业务逻辑层（Business Logic Layer）**：处理核心业务逻辑，包括用例管理、任务调度等
3. **数据访问层（Data Access Layer）**：负责与数据库交互，执行数据的增删改查

## 5. 接口设计

### 5.1 RESTful API

LuckyFrame 提供了 RESTful API 接口，支持第三方系统集成。主要接口包括：

- **测试执行接口**：触发测试任务执行
- **测试结果查询接口**：查询测试执行结果
- **用例管理接口**：管理测试用例
- **计划管理接口**：管理测试计划

### 5.2 内部接口

内部接口主要用于 Web 端与 Client 端之间的通信，基于 Netty 实现。主要功能包括：

- **任务分发**：Web 端向 Client 端分发测试任务
- **执行状态上报**：Client 端向 Web 端上报执行状态
- **日志传输**：Client 端向 Web 端传输执行日志

## 6. 开发规范

### 6.1 代码规范

- 遵循 Java 代码规范，使用驼峰命名法
- 类名使用大驼峰命名法，方法名和变量名使用小驼峰命名法
- 常量使用全大写，单词之间用下划线分隔
- 方法注释使用 Javadoc 格式
- 代码缩进使用 4 个空格
- 每行代码不超过 120 个字符

### 6.2 数据库规范

- 表名使用小写字母，单词之间用下划线分隔
- 字段名使用小写字母，单词之间用下划线分隔
- 主键使用 id 字段，类型为 INT 或 BIGINT
- 外键命名格式为 `表名_字段名`
- 索引命名格式为 `idx_表名_字段名`

### 6.3 命名规范

- 包名使用小写字母，单词之间用点分隔
- 类名使用大驼峰命名法，如 `UserController`
- 方法名使用小驼峰命名法，如 `getUserList`
- 变量名使用小驼峰命名法，如 `userName`
- 常量名使用全大写，单词之间用下划线分隔，如 `MAX_PAGE_SIZE`

1. **数据库配置**：修改 `src/main/resources/application.yml` 文件中的数据库连接信息
2. **服务器配置**：修改 `src/main/resources/application.yml` 文件中的服务器配置
3. **Netty 配置**：修改 `src/main/resources/application.yml` 文件中的 Netty 配置

### 7.3 部署配置

1. **打包**：执行 `mvn clean package` 命令打包项目
2. **部署**：将打包后的 jar 文件部署到服务器
3. **启动**：执行 `java -jar LuckyFrameWeb.jar` 命令启动项目
4. **访问**：通过浏览器访问 `http://服务器IP:端口`

## 8. 开发流程

1. **需求分析**：分析测试需求，确定测试范围和测试策略
2. **用例设计**：设计测试用例，包括接口测试、Web UI 测试、移动端 UI 测试
3. **用例实现**：实现测试用例，包括编写测试脚本、配置测试参数等
4. **测试执行**：执行测试用例，收集测试结果
5. **结果分析**：分析测试结果，生成测试报告
6. **缺陷管理**：管理和跟踪测试过程中发现的缺陷
